# cppseed v0.1.0

This is the first public release of cppseed, a dependency-free C++ CLI for
creating small, buildable CMake projects.

## Highlights

- Generate a CMake project with one command.
- Choose C++17, C++20 (default), or C++23.
- Get a library, executable, and dependency-free CTest test.
- Protect existing paths and clean up partial output after failures.
- Run cppseed without installing Python, Rust, Node.js, or another language
  runtime.

## Requirements

The cppseed executable has no project-specific runtime or template dependency.
The Linux binary requires glibc 2.35 or newer and a compatible system
libstdc++. The Windows binary statically links the MSVC runtime.

Building a generated project requires CMake 3.20 or newer, a compiler that
supports the selected C++ standard, and a build tool supported by CMake.

## Unsigned binary notice

The v0.1.0 binaries are not code-signed or notarized. macOS Gatekeeper or
Windows SmartScreen may display a warning. Verify SHA-256 and ensure the archive
came from `https://github.com/harutasti/cppseed/releases` before proceeding.

## Known limitations

- No package-manager installation yet.
- No user-defined templates or dependency presets.
- Linux musl and ARM64 are not included.
- The macOS deployment target is macOS 13.

Please report bugs through the repository Issues page and security concerns
through the process in `SECURITY.md`.
